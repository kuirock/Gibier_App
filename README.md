# Gibier App (仮称) 🦌🍽️

## 🎯 どんなアプリ？ (What is this?)
北海道のジビエ（エゾシカ肉）に特化した、観光客と地元事業者を繋ぐマッチング・情報発信プラットフォームです。
AIを活用した独自のUI/UXにより、事業者側の「情報を発信したいけれど、ITツールが苦手」という課題を解決します。

## 👥 誰のため？ (Target Audience)
本アプリは、主に以下の2つのユーザー層に向けて設計されています。

1. **北海道を訪れる観光客:**
   美味しいジビエ料理や、地域ならではのディープな食体験・観光体験を求めている人々。
2. **地元の小規模なジビエ事業者・飲食店:**
   こだわりのジビエを提供しているものの、ITリテラシーの壁やリソース不足により、Web上での情報発信にハードルを感じている方々。

## 💡 何のため？ (Purpose)
- **エゾシカ肉の消費拡大と地域活性化:**
  北海道の貴重な観光資源である「ジビエ」の魅力を広め、地域の活性化に繋げるため。
- **情報発信の非対称性をなくす:**
  「美味しいジビエを食べたい観光客」と「魅力的なジビエを提供しているお店」をスムーズに繋ぐため、お店側がストレスなく簡単に店舗情報やメニューを登録できる仕組みを提供します。

## ✨ 主な特徴 (Features)
- **AIアシスト付き入力UI (対話型・フォーム型):**
  LLMを活用し、チャット形式での対話やシンプルなフォーム入力を行うだけで、お店の魅力的な紹介文や登録情報をAIが自動生成・サポートします。これにより、ITツールに不慣れな事業者でも簡単に情報発信が可能になります。
- **直感的な情報検索:**
  観光客が目的のジビエ情報へ直感的にアクセスできる、使いやすいフロントエンド設計。

## 🛠️ 技術スタック (Tech Stack)
- **Frontend:** React, Next.js, TypeScript, Vite
- **Backend/DB:** Supabase
- **AI Integration:** OpenAI API / Gemini API
- **Design/Prototyping:** Figma

# 💡 主要ロジック解説： app/shop/create/page.js
店舗登録画面は、ユーザー体験（UX）を高めるための複雑なフロントエンド制御を行っています。

## 1. ハイブリッド入力UI（チャット＆フォーム）
ユーザーが好みの入力方法を選べるよう、2つのモードをリアルタイムに切り替える制御を行っています。

### 表示切り替え (activeTab)

- 0: チャット形式 / 1: 従来のフォーム形式 としてState管理。

　どちらのモードで入力しても、共通の formData を更新するため、データは常に同期されます。

- ジェスチャー操作（スワイプ）

　モバイル利便性向上のため、onTouchStart / Move / End でフリック方向を計算し、タブの切り替えをトリガーしています。

## 2. 動的なチャット進行システム
SCENARIO 配列に基づくステートマシンとして動作し、ユーザーとの対話を制御します。

### シナリオ定義 (SCENARIO)

- 質問文 (text)、対応する入力項目 (field)、選択肢 (options) を一つの配列で定義。

### インテリジェント・スキップ機能

- 条件分岐 (skipCondition): 特定の回答（例：連絡先不要）に応じて、次の質問を自動でスキップします。

- 入力済み判定 (checkIsFilled): フォーム側ですでに入力された項目がある場合、チャットではその質問を自動的に飛ばし、最短ルートで入力を完了させます。

## 3. 入力アシスト機能（API連携）
ユーザーの入力負荷を軽減するため、外部・内部APIと連携した自動補完ロジックを搭載しています。

### 住所自動入力 (fetchAddress)

- 郵便番号の入力値をフックに zipcloud API を呼び出し、住所データを各フィールドへ自動展開します。

### 店舗サジェスト機能（※調整中）

- 店名入力時（フォーカスアウト時など）に /api/item/search でDB内を検索。

- 既存店舗がヒットした場合にモーダルを表示し、承諾されると住所やカテゴリを一括で流し込みます。

## 4. AIライティング・アシスト (handleAiSuggest)
生成AIを活用し、店舗紹介文のクオリティを底上げする機能です。

### コンテキスト送信

- 店名・業態・ユーザーが書いた下書きを /api/enhance-shop へ送信し、AIが文脈を理解します。

### マルチトーン提案

- AIから返却された「通常・高級感・カジュアル」の3案をカード形式で提示。

- ワンクリックで現在のテキストエリアを上書き（handleSelectAiSuggestion）できるシームレスな操作感を実現しています。

## 5. データ永続化の最終処理 (handleSubmit)
送信直前に、バックエンドのスキーマ仕様に合わせてデータを再構築します。

### データの正規化

- 「姓・名」の結合や、住所パーツの連結、営業時間等の詳細情報をJSON文字列へシリアライズ（description フィールドへ格納）します。

### 認証付きリクエスト

- Cookieまたはヘッダーから取得したJWTトークンを付与し、/api/item/create へセキュアに送信します。
